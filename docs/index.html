A spec in, a whole doc site out

Keep one source of truth in YAML (or reuse your existing OpenAPI 3.x). docs-generator turns it into a single-page site — no build step, hot-reload in dev.

# index.yaml
title: Account Service
sections:
  - name: Auth
    base_url: https://account.example.com
    endpoints:
      - method: POST
        path: /api/v1/auth/login
        body: { email: "", password: "" }
→
  • 📖 Reference — every endpoint, grouped by section
  • 🧪 Live tester — fire requests, creds per environment
  • 🗺️ Diagrams — architecture nodes & edges, auto-laid-out
  • 🔀 Flows & screens — sequences and UI walkthroughs

Up in one command

Scaffold a starter spec, then serve — or drop in a swagger.yaml you already have.

Start fresh

go run ./cmd/server init myspec
go run ./cmd/server \
  -spec ./myspec/index.yaml

Use your OpenAPI

go run ./cmd/server \
  -spec ./swagger.yaml

OpenAPI 3.x is read natively — no conversion.

Ship it

./docs-generator \
  -spec ./spec/index.yaml -port 8080

Single binary, or the published Docker image.

Built for real backends

🧪 Interactive tester

Send requests straight from the docs; per-environment credentials live in localStorage, never in the spec.

🌐 Per-section base URL

Describe a whole platform on one page — account-service and storage-service on different hosts, side by side.

🗺️ Architecture diagrams

ReactFlow nodes & edges with automatic dagre layout when you don't pin positions.

📡 Events & async

Document Kafka topics, webhooks, and pub/sub channels right next to your HTTP endpoints.

🎨 Theming

Override title, logo, primary color, and favicon without touching the template.

🤖 AI-ready endpoints

/docs/spec (JSON), /docs/openapi, and /docs/yaml expose the spec for tooling and agents.
